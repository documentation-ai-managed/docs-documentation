## Project Structure

A good documentation structure starts with clear categories and logical navigation. Use folders and sections to group related content together.

<Tabs>
  <Tab title="API Documentation" icon="code">
    Organize API docs by endpoint type and version.

    | Section | Content |
    |---------|---------|
    | Authentication | Login and token management |
    | Core Endpoints | Main API functionality |
    | Webhooks | Event notifications |
  </Tab>
  <Tab title="Product Guides" icon="book">
    Structure product documentation by user journey.

    | Section | Content |
    |---------|---------|
    | Getting Started | Onboarding and setup |
    | User Guide | Daily usage instructions |
    | Troubleshooting | Common issues and solutions |
  </Tab>
</Tabs>

<Steps>
  <Step title="Plan Your Structure" icon="target">
    Identify your audience and their information needs. Create a sitemap or outline before writing.
  </Step>
  <Step title="Create Categories" icon="folder">
    Use folders and sections to group related topics. Keep navigation levels to 3 or fewer.
  </Step>
  <Step title="Write Content" icon="edit">
    Follow consistent formatting and style guidelines across all pages.
  </Step>
  <Step title="Review and Update" icon="refresh-cw">
    Regularly review content for accuracy and relevance. Update as your product evolves.
  </Step>
</Steps>

<Callout kind="tip">Use consistent terminology throughout your documentation to avoid confusion.</Callout>

## Content Best Practices

Writing effective documentation requires attention to clarity, accuracy, and user experience. Focus on actionable information that helps users achieve their goals.

<Expandable title="Writing Guidelines" default-open="true">
  - Use active voice and present tense
  - Write for your audience's skill level
  - Include practical examples and code snippets
  - Keep sentences and paragraphs concise
  - Use headings and lists for scannability
</Expandable>

<ExpandableGroup>
  <Expandable title="Version Control" default-open="false">
    Use version control to track changes and maintain documentation history. Tag releases and maintain version-specific docs.
  </Expandable>
  <Expandable title="Search Optimization" default-open="false">
    Write descriptive titles and headings. Include keywords users might search for.
  </Expandable>
  <Expandable title="Accessibility" default-open="false">
    Ensure your documentation is accessible to users with disabilities. Use alt text for images and proper heading structure.
  </Expandable>
</ExpandableGroup>
